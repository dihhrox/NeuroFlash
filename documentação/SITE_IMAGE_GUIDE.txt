1. VISAO GERAL

Este arquivo orienta a manutencao dos assets visuais usados no site Neuroflash.
Ele cobre imagens da Home, Quem Somos, FAQ, background compartilhado e icones do rodape.

As dimensoes reais abaixo refletem os arquivos presentes no projeto no momento desta revisao.
As dimensoes ideais sao recomendacoes para futuras substituicoes, buscando nitidez em desktop, mobile e telas de alta densidade.

Quando houver diferenca entre dimensao real e dimensao declarada no HTML, respeitar a observacao especifica do asset. Em alguns casos, a dimensao declarada funciona como placeholder tecnico para reduzir layout shift, nao como tamanho real obrigatorio do arquivo.

6. REGRAS GERAIS DE PRODUCAO

- Usar nomes de arquivo em ASCII, sem acentos, cedilha ou espacos.
- Preferir JPG de alta qualidade para fotos e PNG para icones/elementos com transparencia.
- Manter assunto principal longe das bordas quando o CSS usa object-fit: cover.
- Usar object-fit: contain apenas quando a imagem inteira precisa aparecer, como o packshot.
- Sempre revisar crop em mobile e desktop antes de publicar.
- Declarar width e height no HTML para imagens de conteudo quando possivel.
- Usar loading lazy em imagens abaixo da dobra e decoding async em todas as imagens de conteudo.
- Nao aplicar lazy loading ao background global, pois ele faz parte da identidade visual inicial do site.
- Se uma imagem nova parecer muito fechada, preferir ajustar o arquivo fonte ou object-position antes de alterar a arquitetura do layout.
